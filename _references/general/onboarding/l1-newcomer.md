# L1 — Newcomer (Junior)

> Team members learning fundamentals. Need guidance on *how* and *where*.

## Characteristics

- Limited professional experience in the role (0-2 years)
- Needs explicit step-by-step instructions
- Benefits from structured mentorship and pair work
- Learns best through guided practice, not documentation
- May not recognize when they're stuck or making incorrect assumptions

## Layer 2 — Level-Specific Onboarding Content

### Support Structure

- **Buddy/mentor assignment**: Pair with an L3+ team member. Schedule daily 15-minute check-ins for the first two weeks, then twice weekly for the next month.
- **Pair programming sessions**: Minimum 2 hours per week for the first month, working on real tasks together.
- **Code review mentoring**: First 5 PRs should be reviewed with detailed, educational feedback — not just approval/rejection but explanation of *why*.

### Learning Path

| Week | Focus | Deliverable |
|------|-------|-------------|
| 1 | Environment setup, project overview, first guided task | First commit merged |
| 2 | Conventions deep-dive, second guided task | Second commit with fewer review iterations |
| 3-4 | First independent task (well-scoped, with safety net) | Independent PR with self-review checklist |
| 5-8 | Gradual task complexity increase | Consistent contributions with decreasing review feedback |

### Material Format Preferences

- **Do**: Step-by-step walkthroughs, annotated code examples, video recordings of pair sessions, checklists
- **Don't**: Dense reference documents, architecture decision records (too abstract), large-scope tasks

### AI-Assisted Development Guidance

- Start by using AI to **explore and understand** the codebase: "What does this function do?", "How is this component used?"
- Always **read and understand** AI-generated code before committing — never treat it as a black box
- Use AI to **explain error messages** and suggest fixes, then verify the suggestion makes sense
- Pair AI assistance with mentor review: AI proposes, mentor validates, newcomer learns the *why*
- **Red flag**: If you can't explain what the code does in plain language, don't commit it

### Onboarding Milestones

| Milestone | Target | Verification |
|-----------|--------|-------------|
| Environment running | Day 1 | All tests pass locally |
| First commit | Week 1 | PR merged with mentor approval |
| Follows conventions without reminders | Week 4 | PR passes review with minimal style feedback |
| Independently scopes tasks | Month 2 | Self-assigned task completed without scope changes |
| Mentors another newcomer | Month 6+ | Can explain project patterns to others |
