# TEMPLATE - COMMUNICATION STYLE

> **How to use this template:** Copy this file to `project/communication-style.md` and customize for your project. Replace `{{placeholders}}` with your project's actual values. This file controls the visual style, tone, and framing of all communication materials generated by the `/communication` skill. The `md_to_html.py` script reads this file to produce styled HTML output.

## 2. Tone & Voice

| Setting | Value | Options |
|---------|-------|---------|
| `FORMALITY_LEVEL` | {{FORMALITY_LEVEL}} | `formal` / `semi-formal` / `casual` |
| `TECHNICAL_DEPTH` | {{TECHNICAL_DEPTH}} | `high` / `medium` / `low` |
| `PERSONALITY_TRAITS` | {{PERSONALITY_TRAITS}} | 3-5 comma-separated adjectives (e.g., "precise, approachable, pragmatic") |

**Tone guidance:**

- **Formal**: No contractions, third-person references, structured prose. Suitable for clients and academics.
- **Semi-formal**: Contractions permitted, direct address ("you"), balanced structure. Suitable for evaluators.
- **Casual**: Conversational, first-person plural ("we"), short paragraphs. Suitable for end users.

The per-audience overrides in §6 below can override these defaults for specific audience segments.

## 5. HTML Conversion

| Setting | Value | Description |
|---------|-------|-------------|
| `CONVERSION_ENGINE` | {{CONVERSION_ENGINE}} | `python-markdown` (default) or `pandoc` |
| `PANDOC_OPTIONS` | {{PANDOC_OPTIONS}} | Pandoc CLI flags if pandoc engine selected (e.g., `--toc --toc-depth=3 --highlight-style=tango`) |
| `JINJA2_TEMPLATE_PATH` | {{JINJA2_TEMPLATE_PATH}} | Custom Jinja2 template path (leave empty to use built-in template) |
| `OUTPUT_NAMING` | {{OUTPUT_NAMING}} | Output file naming convention (e.g., `communication-{id}-{audience}.{ext}`) |

### Python-Markdown Extensions

When using the `python-markdown` engine, the following extensions are enabled by default:

- `tables` — pipe-delimited table support
- `fenced_code` — fenced code blocks with language hints
- `toc` — table of contents generation
- `meta` — YAML metadata block parsing

### Pandoc Configuration

When using the `pandoc` engine, the script invokes:

```
pandoc <input.md> --standalone --self-contained {{PANDOC_OPTIONS}} -o <output.html>
```

> **Fallback:** If pandoc is selected but not found on PATH, the script falls back to `python-markdown` with a warning.

## 6. Per-Audience Overrides

Override the default tone and technical depth settings from §2 for specific audience segments. The `/communication` skill applies these overrides when generating material for the matching audience tag.

| Audience Tag | Audience Name | Formality Override | Technical Depth Override | Notes |
|--------------|---------------|--------------------|--------------------------|-------|
| `EVL` | Evaluators | `semi-formal` | `high` | Technical decision-makers; lead with architecture and trade-offs |
| `CLT` | Clients | `formal` | `low` | Product clients; focus on product outcomes, metrics, delivery |
| `USR` | End Users | `casual` | `low` | Non-technical users; plain language, user-centric framing |
| `ACD` | Academics | `formal` | `high` | Researchers; rigorous language, cite theoretical foundations |

> **Note:** If an audience tag is not listed here, the defaults from §2 apply. Add rows for custom audience segments as needed.
