# Journeys

Each journey is a scenario-based starting point that guides you through SEJA adoption for your specific situation. Before diving into a journey, you may want to read the [Concepts](../concepts/) section to understand the key ideas behind the framework -- especially [What Is SEJA](../concepts/what-is-seja.md) and [The Design-Intent Lifecycle](../concepts/design-intent-lifecycle.md).

**New to agentic design?** Read [How to Think About Working with SEJA](../tutorials/designers-mental-model.md) first.

Scenario-based starting points for adopting the SEJA framework. Each journey describes the context, recommended workflow, and key skills for a specific team shape or project situation.

## Solo

| Journey | Scenario |
|---------|----------|
| [Solo Designer -- New Product](journey-solo-designer-greenfield.md) | A product designer starting a brand-new product from scratch. |
| [Solo Designer -- Existing Product](journey-solo-designer-brownfield.md) | A product designer working on an existing product that needs redesign or extension. |
| [Solo Developer -- Side Project](journey-solo-developer.md) | An experienced developer building a side project with architecture-first approach. |

## Team

| Journey | Scenario |
|---------|----------|
| [Small Team -- New Project Kickoff](journey-small-team-kickoff.md) | A small team (2--5 people) bootstrapping a new project together. |
| [Growing Team -- Existing Product](journey-growing-team.md) | A medium team (5--15 people) with specialized roles adopting the framework on an existing product. |

## Multi-project / Special Contexts

| Journey | Scenario |
|---------|----------|
| [Agency -- Multiple Client Projects](journey-agency-multi-project.md) | Agencies or consultancies managing multiple client projects with the workspace pattern. |
| [Enterprise -- Regulated System](journey-enterprise-evolution.md) | Regulated or enterprise domains needing compliance, audit trails, and governance. |
| [Teaching and Research](journey-teaching-research.md) | Educators and researchers using the framework for explanation, analysis, and academic material. |

## How to choose

- **Starting fresh?** Begin with the solo designer (greenfield) or small team kickoff journey.
- **Have an existing system?** Start with solo designer (brownfield) or growing team.
- **Multiple clients?** See the agency journey for the workspace pattern.
- **Need compliance?** The enterprise journey covers audit trails and security reviews.

Each journey links to relevant [recipes](../recipes/) for step-by-step instructions on specific tasks.
