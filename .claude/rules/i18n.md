# i18n Rules

- **Locale codes**: RFC 5646 — `pt-BR`, `en-US`. Never bare `pt` or `en` (except email template filenames).
- **Key parity**: every key in `pt-BR.json` must exist in `en-US.json` and vice versa — same structure, same nesting.
- **Backend .po files**: update both `en_US` and `pt_BR` `.po` files, then `pybabel compile -d translations`.

See `project-i18n-standards.md` for full conventions.
